# 每日温度
[每日温度](https://leetcode.cn/problems/daily-temperatures/?envType=study-plan-v2&envId=top-100-liked)

经典单调栈的题目，其实顾名思义，单调栈就是要维护栈内部的单调性

## 解析

我们设置一个栈来加入待统计的日期，也就是说在栈内的都是result数组还没有值(值为0)的日期

我们先将第0天加入，因为它一定是待统计的日期。
接下来我们从第一天开始看，如果第一天比栈顶(也就是第0天)热，说明第0天的值找到了,也就是i-st.top()
弹出第0天并加入第1天

后面以此类推，我们检查到第i天时，如果小于等于栈顶温度，说明栈顶和第i天都还需要等待新的更高温度出现
如果比栈顶温度大，弹出栈顶直到栈空或者遇到栈中更高温度，这些日期都找到结果所以弹出，然后再加入第i天

怎么样，不算难吧，重点是一定要维护单调栈的单调性
单调栈一定要单调！

## 代码
```
class Solution {
public:
    vector<int> dailyTemperatures(vector<int>& temperatures) {
        //本题使用单调栈完成
        //思路其实很好想，我们设置的栈中只能是栈底到栈顶的单调递减
        //主打一个秋后算账，等到有更大温度的值把前面的小值清算了
        stack<int> st; //存储的是下标,这样我们既可以获取温度也可以得知日期
        st.push(0);//第一个值没有前面的日期和它比较
        vector<int> result(temperatures.size(),0);
        //i表示日期，各位应该知道的
        for(int i=1;i<temperatures.size();i++)
        {
            if(st.empty() || temperatures[i] <=temperatures[st.top()])
                st.push(i);
            else
            {
                while(!st.empty()&&temperatures[i] >temperatures[st.top()])
                {
                    result[st.top()]=i-st.top();
                    st.pop();
                }
                st.push(i);//然后还要加入该日期
            }
        }
        
        return result;

    }
};
```

时间复杂度O(n)
空间复杂度O(n)
