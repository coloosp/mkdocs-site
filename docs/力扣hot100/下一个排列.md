## 解析
相当有趣的题目，也可以说相当具有技巧性
各位思考一下给定一些数字，比如1,2,3,4,5
如何组合让它们最小，又怎样是最大
答案是升序的12345最小，降序的54321最大

我们知道了降序大于升序，这很合理，并且对于比如
12345<12543没问题吧，这个道理在局部也能体现

比如说对于13542，它的下一个排列是什么？
应该是一个比它更"降序的序列'
我们看到542已经是降序了，所以对于542不能做出更改，但直到3542却有3这个3->5的升序。
我们应该对这个3下手，请记住，只能**稍微大一点点**，我们应该把542中比3大的4拿过来，变成14532
这样就没问题了吗，不对，有在这两个数之间的值
也就是说在4后面如果是升序的话会更小，也就是14235

我们的思路是这样的
* 从右向左找，直到找到一个不和谐的家伙，也就是[i,n-1]不表现为降序时，我们对这个i下手
* 在[i+1,n-1]里面找到一个比nums[i]大一点点的值，也就是确保比nums[i]大并最小，称为nums[index],我们交换两者
* 然后将[i+1，n-1]排序为降序
* 特殊情况是i=-1,说明整个数组都是降序的，这时候会回归起点，也就是升序是下一个排列，我们翻转数组即可

## 代码
```
class Solution {
public:
    void nextPermutation(vector<int>& nums) {
        //稍微大一点点
        int n=nums.size();

        int i=n-2;
        while(i>=0 && nums[i] >=nums[i+1]) //找到第一个导致[i,n-1]不是降序的值
            i--;

        if(i>=0)
        {
            int index=i+1;
            int curnum=nums[i+1];
            for(int j=i+1;j<n;j++) //找到一个比nums[i]稍微大一点的值
            {
                if(nums[j]>nums[i] && nums[j] <curnum)
                    {
                        index=j;
                        curnum=nums[j];
                    }
            }

            swap(nums[i],nums[index]);//交换两者
            sort(nums.begin()+i+1,nums.end()); //将后续升序排列保证最小
            return ;
        }

        reverse(nums.begin(),nums.end()); //如果i==-1说明原来就是降序数组，翻转数组为升序

    }
};
```

时间复杂度O(nlogn)
