# 环形链表
依旧是使用快慢指针来实现

说实话写到现在的链表题，因为都是过于经典的数据结构题目，所以反而我不知道怎么去说明了
抱歉各位这是我的问题，我在学习链表的知识到现在只留下了偏向于直觉的做法，也就是"这道题本来就是这样做的"这种感觉。

我就不多废话了，快慢指针能做这道题的原因在于如果有环，快指针一次走两格，慢指针一次走一格，它们一定还会再相遇。

各位也可以尝试在纸上试试是否是这样的

```
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode(int x) : val(x), next(NULL) {}
 * };
 */
class Solution {
public:
    bool hasCycle(ListNode *head) {
        //依旧是快慢指针
        ListNode *fast =head;
        ListNode *slow=head;
        while(fast!=NULL){
            slow=slow->next;
            fast=fast->next;
                if(fast==NULL) //这是为了避免快指针提前走入空节点
                    return false;
            fast=fast->next;
            if(fast==slow)
                return true;
        }

        return false;

    }
};
```

时间复杂度O(n)
空间复杂度O(1)

当然你可以设置一个ListNode*的set集合，当再次遇到时就说明有环，这也是很经典的做法
