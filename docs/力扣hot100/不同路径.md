# 不同路径
[不同路径](https://leetcode.cn/problems/unique-paths/description/?envType=study-plan-v2&envId=top-100-liked)

结束了动态规划的章节，欢迎来到多维动态规划(耍我呢)，多维动态规划主要和二维的路径有关，思想应该是动态规划的增强

## 解析
不过这一道题很简单了，可以说是二维爬楼梯了
各位想象爬楼梯的状态转移方程，应该是当前楼梯i的方法数=前面一级i-1楼梯的方法数+前面两级i-2楼梯的方法数
这里也很好想，当前节点[i][j]的路径数=上方[i-1][j]的方法数+左边[i][j-1]的方法数
因为我们的故障机器人只能往下和往右
不过各位还需要注意一些边界问题

我们的dp数组预留了第0行和第0列避免了一些超界问题，各位根据实际情况实现
1. **确定dp数组与下标的含义**:dp[i][j]为到达i,j点的方法数
2. **确定递推公式**:dp[i][j]=dp[i-1][j]+dp[i][j-1]
3. **dp数组初始化**:dp[1][1]=1
4. **确定遍历顺序** :先遍历行再遍历列，都从1开始，除了dp[1][1]不需要更新之外
5. **举例推导dp数组**
比如一个2*3的数组
|  0  |  0  |  0  |
| --- | --- | --- |
| 0   | 1   | 1   |
| 0   | 1   | 2   |
| 0   | 1   | 3   |  
  
## 代码
```
class Solution {
public:
    int uniquePaths(int m, int n) {
        vector<vector<int>> dp(m+1,vector<int>(n+1,0) );//预留第0列和第0行
            dp[1][1]=1;//初始化出发点
        for(int i=1;i<=m;i++)
        {
            for(int j=1;j<=n;j++){
                if(i==1&&j==1) //除了dp[1][1]不需要更新
                    continue;
                dp[i][j]=dp[i-1][j]+dp[i][j-1];
            }

        } 

        return dp[m][n];

    }
};
```

时间复杂度O($n^2$)
空间复杂度O($n^2$)
