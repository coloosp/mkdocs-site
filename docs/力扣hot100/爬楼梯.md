# 爬楼梯
[爬楼梯](https://leetcode.cn/problems/climbing-stairs/description/?envType=study-plan-v2&envId=top-100-liked)
欢迎进入动态规划的章节，动态规划绝对也是相当重要的思想，基本上算法题必出现的题型。对于动态规划，作者的理解是动态规划是存在最优子结构，根据前面的子结构来更新现有问题，存在状态更新方程。这可能类似于我们前面解决的递归，将一个问题拆分成几个类似的子问题递归解决。但动态规划是自底向上的思维，是从子结构更新到总体的思维方式，递归则是自顶向下的模式。

对于动态规划各位可以安装以下步骤完成  
1. **确定dp数组与下标的含义**
2. **确定递推公式**
3. **dp数组初始化**
4. **确定遍历顺序** 
5. **举例推导dp数组**

## 解析
好的话不多说，进入本节问题，还是相当简单的，各位应该可以用各种方式解决

实际上比如我们爬n级楼梯的方法数为爬n-1级楼梯的方法数+爬n-2级楼梯的方法数之和。
如果从0开始计算的话序列应该是
1 1 2 3 5 8...
各位可以发现是斐波那契数列，所以其实可以用递归或者记忆化递归来解决，各位也可以自行尝试

我们试着按照动态规划五步来实现一下这道简单题吧
1. 确定dp数组与下标的含义:dp[i]表示爬i级楼梯需要的方法数
2. 确定递归公式：dp[i]=dp[i-1]+dp[i-2]
3. dp数组初始化，这里需要将dp[0]=1,这是特殊情况，因为要考虑到dp[2]=dp[0]+dp[1]，并且dp[1]=1
4. 确定遍历顺序：从2级楼梯开始向上探索
5. 举例推导dp数组：如上所示

### 代码
```
class Solution {
public:
    int climbStairs(int n) {
        vector <int> dp(n+1,0); //dp数组，存储到达dp[i]的方法数
        dp[0]=1;
        dp[1]=1;
        for(int i=2;i<=n;i++)
        {
            dp[i]=dp[i-1]+dp[i-2]; //到达dp[i]的方法数为前面1级和前面2级台阶的方法数之和
        }

        return dp[n];

    }
};
```
时间复杂度O(n)
空间复杂度O(n)

其实有更省空间的方式，注意到我们只需要保存前面两级楼梯的方法数，所以其实不需要所谓dp数组
```
class Solution {
public:
    int climbStairs(int n) {

        if(n==1) //特判
            return 1;

        int pre1=1;//前2级楼梯
        int pre2=1;//前1级楼梯

        int cur;

        for(int i=2;i<=n;i++)
        {
            cur=pre1+pre2;
            pre1=pre2; //更新一级
            pre2=cur; //更新一级
        }

        return cur;




    }
};
```
怎么样，很简洁吧
时间复杂度O(n)
空间复杂度O(1)
