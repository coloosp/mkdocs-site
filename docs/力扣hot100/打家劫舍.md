## 解析

话说回来，这其实是偷与不偷的选择，如果我们偷了这家，我们就不能偷上一家，但直到上上一家偷到的钱还是没有问题的。如果我们不选择偷这家，那直到上一家为止偷到的钱也是没有问题的

问题就在判断两者的大小，这也是动态规划很常见的选择形式，做还是不做呢？这是个问题

其实看完上面的描述各位可能想到了状态更新公式
dp[i]=max(dp[i-2]+nums[i],dp[i-1]);

1. **确定dp数组与下标的含义**：dp[i]表示直到i家为止能偷到的最大金额，注意这不表示一定要偷第i家
2. **确定递推公式**：dp[i]=max(dp[i-2]+nums[i],dp[i-1]);
3. **dp数组初始化**:dp[0]=nums[0],dp[1]=max(nums[0],nums[1])
4. **确定遍历顺序** :从第一家偷起
5. **举例推导dp数组**：  
例如[1,2,3,1]
dp[0]=1 
dp[1]=max(1,2)=2
dp[2]=max(2,1+3)=4
dp[3]=max(2+1,4)=4


## 代码
```
class Solution {
public:
    int rob(vector<int>& nums) {
        //偷与不偷，这是个问题
        //其实很简单，如果我们决定要偷这家的，就不能选择上一家的，否则前按照上一家的来算
        if(nums.size()==1)
            return nums[0];//只有一家那还说啥

        vector<int> dp(nums.size()); //dp[i]表示直到第i家能偷到的最大金额
        dp[0]=nums[0];
        dp[1]=max(nums[0],nums[1]);
        for(int i=2;i<nums.size();i++)
        {
            dp[i]=max(nums[i]+dp[i-2],dp[i-1]);//要么偷这家，我们选择这家和前前加能偷到的最大金额，要么不偷这家，我们选择前一家能偷到的最大金额
        }

        return dp[nums.size()-1];
    }
};
```

时间复杂度O(n)
空间复杂度O(n)
