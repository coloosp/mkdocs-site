我说一个粗浅且不一定对的结论，这里使用了后序遍历的思维，并且会利用后序遍历的左右值来总结

再看看我们动态规划的思维，利用子问题的结果来获取更高一级问题的结果，换句话说我们在做状态转移和最优子结构的计算

作者也有很多需要提升的，会更多学习动态规划的知识

说回这道题来，对于最大路径和该怎么计算？不要被例子骗了，我们回想一下二叉树的直径思想，对于一点的最大路径和=左子树最大路径和+右子树最大路径和+当前节点值(只不就是状态转移方程吗)
左右子树最大路径和也是递归定义的，最终会回到叶子节点处，并且只有结果为正时我们才会加入，否则对我们的总体是负面影响可以直接排除

在找到一个节点的最大路径和时，我们会更新result

返回值是这段代码最精妙的地方，想想我们要获取什么样的最优子结构，答案是我们需要返回包含该点的最佳路径，注意这里的路径并非要包含到叶子节点

```
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode() : val(0), left(nullptr), right(nullptr) {}
 *     TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
 *     TreeNode(int x, TreeNode *left, TreeNode *right) : val(x), left(left), right(right) {}
 * };
 */
class Solution {
public:
    int result=INT_MIN;
    
    //返回值为包含该节点的最优路径和
    int dfs(TreeNode* node)
    {
        if(node==nullptr)
            return 0;
        
        int leftval=max(dfs(node->left),0);//只有加入正值才对整体结果有收益
        int rightval=max(dfs(node->right),0);

        int curval=node->val+leftval+rightval;
        result=max(result,curval);

        return node->val+max(leftval,rightval); //这是最关键的代码，我们只会返回一条最优路径
    }
    int maxPathSum(TreeNode* root) {
        dfs(root);
        return result;
    }
};
```

时间复杂度O(n)
空间复杂度O(n)
