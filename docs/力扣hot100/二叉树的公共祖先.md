# 二叉树的公共祖先
[二叉树的公共祖先](https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-tree/?envType=study-plan-v2&envId=top-100-liked)

这道题作者应该是第三次写了，各位初见的话不一定能想清楚为什么要这么写

首先给出结论，这里我们用到了后序遍历

我想想该怎么解释，这样吧，比如力扣探险队需要寻找丢失的两件宝物p和q，当它们汇集到一点（也就是最近公共祖先）时会发挥出真正的力量，我们的计划是这样的，兵分两路各自报点，看看左路(左子树)和右路(右子树)是否是藏有宝物的路径，由于二叉树的特性，我们的搜索往往是递归定义的，也就是我们会一直兵分两路下去
我们约定以下内容
1. 如果走到尽头(空节点)，向上一级汇报这里为空
2. 如果找到宝物pq，向上一级汇报本地点
3. 如果不符合上述情况，当前节点的队长观察它的左路和右路情况，如果有任意一路发现了宝物，向上级汇报宝物路径节点，如果都没有则返回空，如果两边都有宝物，说明要找的就是该点，直接返回该点

从根节点开始，所有队长层层执行这套规则，最终根节点的汇报结果，就是两件宝物 p 和 q 的 “最近公共祖先”

前面的小故事只是我瞎编的，如果困扰了大家请不要尝试再理解

因为这道题属于经典题，各位无法理解的话就多看看代码理解原理

```
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode(int x) : val(x), left(NULL), right(NULL) {}
 * };
 */
class Solution {
public:
    TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
        if(root==NULL || root==p || root==q) //有三种情况，遇到空，或者找到节点都返回本身
            return root;
        TreeNode *leftnode =lowestCommonAncestor(root->left,p,q); //查找左子树有没有我们需要的节点
        TreeNode *rightnode=lowestCommonAncestor(root->right,p,q);
        if(leftnode && rightnode) //左右节点都查找到了节点，说明当前节点就是最近公共祖先
            return root;   
        if(leftnode==NULL && rightnode)//左边没有查找到节点而右边查找到节点，说明这里处于右端
            return rightnode;
        if(rightnode== NULL && leftnode)
            return leftnode;

        return NULL; //说明以该节点为根的树没有我们需要的节点
        
    }
};
```

时间复杂度O(n)
空间复杂度O(H) 为递归栈开销,H为二叉树高度
