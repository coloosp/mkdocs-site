# 随机链表的复制
[随机链表的复制](https://leetcode.cn/problems/copy-list-with-random-pointer/description/?envType=study-plan-v2&envId=top-100-liked)

理解清楚题意并完成不是很简单，简单来说就是要构造一个和题目链表一模一样的链表


## 回溯+哈希表
如果按照next与random顺序构造会遇到一个问题，random是随机节点，因为random可能会指向重复节点，所以根据random线性构造是不可取的，这要求我们需要使用哈希表来保证该节点是否被构造过

哈希表的结构应该为[原链表节点指针，新链表节点指针]，使用unorderd_map来实现。

我们使用了回溯+哈希表的思想来构造链表，这也是力扣的标准题解，回溯与递归的思想可以很好解决构造顺序的问题
```
/*
// Definition for a Node.
class Node {
public:
    int val;
    Node* next;
    Node* random;
    
    Node(int _val) {
        val = _val;
        next = NULL;
        random = NULL;
    }
};
*/

class Solution {
public:
    //使用了哈希表加回溯的思想实现
    unordered_map<Node*,Node*> hashNode;
    Node* backtraking(Node* node)
    {
        if(node==NULL)
            return NULL;

        if(!hashNode.count(node)) //当哈希表中没有该节点，说明需要构造
        {
            Node *newNode =new Node(node->val);//必须有值参数的构造函数
            hashNode[node]=newNode;//原链表节点映射新链表节点
            newNode->next=backtraking(node->next);
            newNode->random=backtraking(node->random);
        }
        return hashNode[node];//返回构造好的或者已经存在的节点
    }


    Node* copyRandomList(Node* head) {
        return backtraking(head);
    }
};
```
时间复杂度O(N)
空间复杂度O(N)

## 迭代+哈希表
或者说真的需要麻烦的回溯吗？
这是在力扣上看到大佬写的，因为我们使用了哈希表存储的节点，记住我们哈希表的特性就是通过原链表节点找到新链表节点，也就是说只要使用哈希表就不会存在找不到接下来节点的情况，我们实际上可以这样写

```
/*
// Definition for a Node.
class Node {
public:
    int val;
    Node* next;
    Node* random;
    
    Node(int _val) {
        val = _val;
        next = NULL;
        random = NULL;
    }
};
*/

class Solution {
public:
    Node* copyRandomList(Node* head) {
        unordered_map<Node*,Node*> hashNode;
        for(Node* p=head;p!=NULL;p=p->next) //构造节点
        {
            hashNode[p]=new Node(p->val);
        }

        for(Node* p=head;p!=NULL;p=p->next) //利用哈希表中存储节点完成next和random
        {
            hashNode[p]->next=hashNode[p->next];
            hashNode[p]->random=hashNode[p->random];
        }

        return hashNode[head];
    }
};
```
