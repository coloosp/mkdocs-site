# 单词拆分
[单词拆分](https://leetcode.cn/problems/word-break/?envType=study-plan-v2&envId=top-100-liked)

我讨厌字符串题(╬▔皿▔)

## 解析
如果各位仔细理解会发现和前面两题的思维模式其实是一致的
为何这样说，我们的完全平方数是这样的 当前数字可以被拆分为一个平方数+剩余数字
零钱兑换是这样的 当前面额可以被拆分为一个硬币+剩下面额

为什么可以这样拆分？因为一个硬币/一个平方数是我们的物品之一，这本来就是存在的。**而剩下的部分在前面我们已经计算过了！** 因为动态规划是**自底向上**的思维

那这道题就是 直到当前为止 比如[0,i)范围的字符串可以被拆分为 在wordDict中有的一个字符串+剩下的字符串

思考我们的dp数组应该是什么形式的，应该是bool类型的，标记当前下标i之前的字符串都是符合条件可以匹配的
就算我们想出来了，字符串的题也不是好做的，字符串题就是想出来不一定容易，做起来一定不容易

因为案例中给我们的只有vector 我们要转换为set来方便匹配

还有一个问题，比如我们用Apple匹配了Apple，应该判断dp[0]是否正确，所以我们要把dp[0]=true,dp数组要向前挪动一位

对于如何切割字符串我们需要使用substr函数别忘了它的参数，并且使用set的匹配功能

1. **确定dp数组与下标的含义**：dp[i]表示下标i之前的字符串匹配情况
2. **确定递推公式**：根据dp[j]与[j,i)来判断是否dp[i]=true,你也可以写成dp[i]=dp[j]&&（当前字符串是否匹配成功）
3. **dp数组初始化** :dp[0]=true,其他的为false
4. **确定遍历顺序** :先遍历背包(字符串下标i)，再遍历物品(根据(j,i]切割字符串)
5. **举例推导dp数组**

## 代码
```
class Solution {
public:
    bool wordBreak(string s, vector<string>& wordDict) {
        //我恨字符串
        //本质完全背包，但是字符串可恶
        vector<bool> dp(s.size()+1,false); //dp[i]表示在下标i以前(不包括i)的单词是否匹配,也就是[0,i)部分
        unordered_set <string> wordset(wordDict.begin(),wordDict.end());//复制一份set方便查找
        dp[0]=true;
        for(int i=1;i<=s.size();i++)
        {
            for(int j=0;j<i;j++)
            {
                string word=s.substr(j,i-j);
                //检查[0,j）部分与[j,i)部分是否符合
                if(dp[j]==true && wordset.find(word)!=wordset.end())
                {
                    dp[i]=true;
                }
            }
        }

        return dp[s.size()];
    }
};
```

时间复杂度：O($n^3$) 这是因为substr也需要花费O(n)的事件
空间复杂度O(n)
